# 16 - 配置系统

## 概述

Claude Code 采用**5 层配置系统**，从用户级到企业策略级逐层覆盖，确保灵活性与安全性的平衡。

- **配置加载**：`utils/settings/settings.ts`
- **类型定义**：`utils/settings/types.ts`
- **常量定义**：`utils/settings/constants.ts`
- **全局配置**：`utils/config.ts`

## 5 层配置架构

```
优先级（低 → 高）：

┌─────────────────────────────────────────┐
│  ① userSettings（用户设置）              │
│     ~/.claude/settings.json              │
│     用户个人偏好，跨项目生效              │
├─────────────────────────────────────────┤
│  ② projectSettings（项目设置）           │
│     .claude/settings.json                │
│     项目共享配置，提交到 Git              │
├─────────────────────────────────────────┤
│  ③ localSettings（本地设置）             │
│     .claude/settings.local.json          │
│     项目本地覆盖，Git 忽略               │
├─────────────────────────────────────────┤
│  ④ flagSettings（标志设置）              │
│     --settings CLI 参数 / SDK 内联       │
│     临时覆盖                             │
├─────────────────────────────────────────┤
│  ⑤ policySettings（策略设置）            │  ← 最高优先级
│     企业托管 / MDM / 远程 API            │
│     管理员强制策略                        │
└─────────────────────────────────────────┘
```

## 文件位置

| 来源 | 路径 | 说明 |
|------|------|------|
| 用户 | `~/.claude/settings.json` | 全局用户设置 |
| 项目 | `.claude/settings.json` | 项目共享设置 |
| 本地 | `.claude/settings.local.json` | 项目本地覆盖 |
| 标志 | `--settings <path>` | CLI 参数指定 |
| 策略 | 平台相关（见下文） | 企业管理 |

### 策略设置加载顺序（先匹配优先）

```
1. 远程 API
   └── getRemoteManagedSettingsSyncFromCache()

2. MDM（移动设备管理）
   ├── Windows: HKLM 注册表
   └── macOS: plist 文件
   └── getMdmSettings()

3. 文件系统
   ├── managed-settings.json
   └── managed-settings.d/*.json  ← 按字母序合并
   └── 路径：平台相关

4. HKCU（Windows 用户注册表）
   └── getHkcuSettings()
```

## 配置安全

### 企业管控

```
企业管理员可以：

1. 通过 policySettings 强制设定
   - 覆盖所有其他来源
   - 不可被用户修改

2. 通过 allowManagedPermissionRulesOnly
   - 限制用户只能使用托管权限规则

3. 通过 allowManagedHooksOnly
   - 限制用户只能使用托管 Hook

4. 通过 allowManagedMcpServersOnly
   - 限制用户只能使用托管 MCP 服务器

5. 通过 strictPluginOnlyCustomization
   - 限制非插件自定义
```

### 验证

所有设置通过 **Zod Schema** 验证：

```typescript
// settings/types.ts
```
