# 14 - 插件与技能

## 概述

Claude Code 提供**插件系统**和**技能系统**两套扩展机制。插件是完整的扩展包（含 Hook、MCP 服务器等），技能是轻量的 Prompt 命令。

## 技能系统 (Skills)

### 什么是技能？

技能是一种特殊的 PromptCommand，通过 Markdown 文件定义，包含 YAML 前端事项和 Prompt 内容。用户通过 `/技能名` 调用。

### 内置技能 (Bundled Skills)

```typescript
// skills/bundledSkills.ts
type BundledSkillDefinition = {
  name: string                    // 技能名
  description: string             // 描述
  aliases?: string[]              // 别名
  whenToUse?: string              // AI 何时使用
  argumentHint?: string           // 参数提示
  allowedTools?: string[]         // 允许的工具
  model?: string                  // 模型覆盖
  disableModelInvocation?: boolean
  userInvocable?: boolean
  isEnabled?: () => boolean       // 启用条件
  hooks?: HooksSettings           // 自定义 Hook
  context?: 'inline' | 'fork'    // 执行上下文
  agent?: string                  // Agent 名称
  files?: Record<string, string>  // 参考文件

  getPromptForCommand(args, context)
    : Promise<ContentBlockParam[]>  // 生成 Prompt
}

function registerBundledSkill(definition: BundledSkillDefinition): void
```

#### 技能文件格式

```markdown
---
name: my-skill
description: 我的自定义技能
aliases:
  - ms
allowedTools:
  - Bash
  - FileRead
model: sonnet
hooks:
  PreToolUse:
    - matcher: "Write"
      hooks:
        - type: command
          command: "echo 'writing...'"
---

你是一个专注于 {{args}} 的助手。

请按照以下步骤操作：
1. 分析需求
2. 实施方案
3. 验证结果
```

### 市场插件

#### 安装流程

```
1. 搜索市场 → 找到插件
     │
2. 验证插件（validatePlugin）
     │  • 检查黑名单（pluginBlocklist）
     │  • 检查策略（pluginPolicy）
     │  • 验证签名
     │
3. 解析依赖（dependencyResolver）
     │
4. 下载并安装
     │
5. 注册到设置（enabledPlugins）
     │
6. 加载插件内容
     │  • loadPluginCommands — 加载命令
     │  • loadPluginHooks — 加载 Hook
     │  • loadPluginAgents — 加载 Agent
     │
7. 初始化 MCP 服务器（如有）
```

#### 插件 ID 格式

```
{pluginName}@{marketplaceName}

示例：
  formatter@anthropic-tools
  linter@community-plugins
  database@builtin
```

## 技能 vs 插件对比

| 特性 | 技能 | 插件 |
|------|------|------|
| 复杂度 | 低（单个 Markdown 文件） | 高（完整包） |
| 能力 | 仅 Prompt 命令 | Hook + MCP + 命令 + Agent |
| 分发 | 文件/目录 | 市场/内置 |
| 企业管理 | 通过设置路径 | 通过策略 |
| 版本管理 | 无 | 有 |
| 依赖 | 无 | 支持 |
| 自动更新 | 无 | 支持 |
