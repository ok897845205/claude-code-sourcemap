## 核心模块划分

### 1. 入口层

| 文件 | 职责 |
|------|------|
| `main.tsx` | CLI 启动入口，初始化序列编排 |
| `setup.ts` | 会话初始化（Git 上下文、终端备份） |
| `entrypoints/init.ts` | 异步初始化（配置验证、遥测） |
| `entrypoints/cli.tsx` | CLI 入口组件 |
| `entrypoints/mcp.ts` | MCP Server 模式入口 |

### 2. 核心引擎

| 模块 | 文件 | 职责 |
|------|------|------|
| 工具系统 | `Tool.ts`, `tools.ts`, `tools/` | 40+ 可执行工具 |
| 命令系统 | `commands.ts`, `commands/` | 80+ 用户命令 |
| 查询引擎 | `QueryEngine.ts`, `query/` | 语义代码搜索 |
| 任务管理 | `Task.ts`, `tasks.ts`, `tasks/` | 后台任务调度 |

### 3. 服务层

| 服务 | 目录 | 职责 |
|------|------|------|
| API 客户端 | `services/api/` | Anthropic API 调用 |
| MCP | `services/mcp/` | Model Context Protocol |
| 分析 | `services/analytics/` | 遥测与特性开关 |
| OAuth | `services/oauth/` | 认证流程 |
| 会话存储 | `services/SessionMemory/` | 持久化上下文 |
| 插件 | `services/plugins/` | 插件发现与管理 |

### 4. UI 层

| 模块 | 目录 | 职责 |
|------|------|------|
| 组件 | `components/` | 150+ React/Ink 组件 |
| 状态 | `state/` | React Context + Store |
| 上下文 | `context/` | 通知、语音、模态框等 |
| 设计系统 | `components/design-system/` | UI 基元 |

## 数据流

### API 请求流程

```
用户输入
  → PromptInput 组件
    → 消息队列
      → Anthropic API (messages.create)
        → 流式响应解析
          → Tool Use 检测
            → 权限检查 (checkPermissions)
              → Allow: 执行工具
              → Ask: 弹出权限对话框
              → Deny: 向模型返回拒绝信息
            → 工具执行 (tool.call)
              → 结果返回给模型
                → 继续对话循环
```

### 工具执行流程

```
模型输出 tool_use block
  ↓
tools.ts: 查找已注册工具
  ↓
Tool.validateInput(): 输入验证（Zod Schema）
  ↓
Tool.checkPermissions(): 权限检查
  ├─ allow → 直接执行
  ├─ ask → 向用户请求许可
  │       ├─ 用户允许 → 执行
  │       └─ 用户拒绝 → 返回拒绝消息
  └─ deny → 返回拒绝消息
  ↓
Tool.call(): 执行工具逻辑
  ├─ onProgress(): 进度回调 → UI 更新
  └─ return ToolResult
  ↓
mapToolResultToToolResultBlockParam(): 序列化结果
  ↓
作为 tool_result 消息发送回模型
```

## 设计模式

### 1. 工厂模式 — buildTool()

所有工具通过 `buildTool()` 工厂函数创建，提供统一的默认值和类型安全：

```typescript
const MyTool = buildTool({
  name: 'my_tool',
  inputSchema: z.strictObject({ ... }),
  async call(args, context) { ... },
  async checkPermissions(input, context) { ... },
  async description(input) { ... },
  async prompt() { ... },
  // ... 其他方法有默认实现
});
```

### 2. 注册表模式 — 工具/命令注册

```typescript
// tools.ts — 所有工具在一处注册
function getAllBaseTools() {
  return [
    BashTool,
    FileReadTool,
    FileEditTool,
    // ... 条件注册
    ...(feature('PROACTIVE') ? [SleepTool] : []),
  ].filter(Boolean);
}
```

### 3. React Context + Store 模式

```typescript
// 自定义 Store（非 Redux）
type Store<T> = {
  getState: () => T
  setState: (updater: (prev: T) => T) => void
  subscribe: (listener: Listener) => () => void
}

// 通过 Context 分发
<AppStoreContext.Provider value={store}>
  {children}
</AppStoreContext.Provider>
```

### 4. 特性开关模式（编译时消除）

```typescript
import { feature } from 'bun:bundle'

// 编译时：内部构建保留代码，外部构建移除
if (feature('BUDDY')) {
  // 伴侣精灵相关代码
}
```

### 5. 延迟加载模式

```typescript
// 大型工具延迟加载，减少初始加载时间
const AgentTool = feature('AGENT_TOOL') 
  ? require('./tools/AgentTool').AgentTool 
  : null;

// 命令延迟加载
const configCommand: Command = {
  type: 'local-jsx',
  load: () => import('./commands/config/index.js'),
}
```

### 6. 分层权限模式

```
全局否决规则 (deny rules)
  ↓ 过滤
工具级输入验证 (validateInput)
  ↓ 验证
工具级权限检查 (checkPermissions)
  ↓ 决策
权限模式覆盖 (PermissionMode)
  ├─ default: 正常流程
  ├─ acceptEdits: 编辑类自动允许
  ├─ plan: 一律拒绝（计划模式）
  ├─ auto: 分类器预测
  └─ bypassPermissions: 全部允许
```

## 并发模型

Claude Code 使用 Node.js 单线程事件循环：

- **工具并发安全标记**：每个工具通过 `isConcurrencySafe()` 声明能否并行执行
  - 安全：FileRead、Grep、Glob、WebFetch 等只读工具
  - 不安全：Bash、FileEdit、FileWrite 等有副作用的工具
- **消息队列**：`messageQueueManager.ts` 管理消息批处理
- **后台任务**：`Task.ts` 支持后台进程（如 `run_in_background`）
- **中断行为**：
  - `cancel`：BashTool — 立即终止进程
  - `block`：默认 — 等待完成
