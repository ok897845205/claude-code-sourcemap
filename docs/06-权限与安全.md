# 06 - 权限与安全

## 概述

权限系统是 Claude Code 的**安全核心**，控制 AI 能做什么、不能做什么。每次工具调用前都会经过权限系统的检查，确保操作安全合规。

- **类型定义**：`types/permissions.ts`
- **核心逻辑**：`utils/permissions/`（23+ 个文件）
- **权限模式**：`utils/permissions/PermissionMode.ts`
- **路径验证**：`utils/permissions/pathValidation.ts`
- **Shell 规则**：`utils/permissions/shellRuleMatching.ts`

## 权限模式

### 六种模式

| 模式 | 标题 | 符号 | 行为 |
|------|------|------|------|
| `default` | Default | — | 每次工具使用都需要用户确认 |
| `plan` | Plan Mode | ⏸ | 只读模式，Claude 只能规划不能执行 |
| `acceptEdits` | Accept Edits | ⏵⏵ | 自动接受文件编辑，其他仍需确认 |
| `bypassPermissions` | Bypass | ⏵⏵ | 跳过所有权限检查（危险！） |
| `dontAsk` | Don't Ask | ⏵⏵ | 不询问直接拒绝未授权操作 |
| `auto` | Auto Mode | ⏵⏵ | 使用 AI 分类器自动判断（需门控） |

### 模式切换顺序

```
          普通用户
┌──────────────────────────────┐
│ default ─► acceptEdits ─► plan ─► bypassPermissions ─► auto ─► default
└──────────────────────────────┘

          内部用户
┌──────────────────────────────┐
│ default ─► bypassPermissions ─► auto ─► default
└──────────────────────────────┘
```

### 模式特性对比

| 特性 | default | plan | acceptEdits | bypass | auto |
|------|---------|------|-------------|--------|------|
| 文件读取 | 🟢 | 🟢 | 🟢 | 🟢 | 🟢 |
| 文件编辑 | ❓询问 | 🔴 | 🟢 | 🟢 | 🤖AI |
| Bash 命令 | ❓询问 | 🔴只读 | ❓询问 | 🟢 | 🤖AI |
| 网络访问 | ❓询问 | 🔴 | ❓询问 | 🟢 | 🤖AI |
| Agent 派生 | ❓询问 | 🔴 | ❓询问 | 🟢 | 🤖AI |

## 权限规则

### 规则结构

```typescript
type PermissionRule = {
  source: PermissionRuleSource    // 规则来源
  ruleBehavior: PermissionBehavior  // allow / deny / ask
  ruleValue: PermissionRuleValue    // 工具名 + 可选内容
}

type PermissionRuleValue = {
  toolName: string       // 如 "Bash", "FileEdit", "mcp__server1"
  ruleContent?: string   // 如 "git status:*" 或 "/path/**"
}
```

### 规则来源优先级

```
policySettings    ←── 企业策略（最高优先级）
    │
flagSettings      ←── 特性开关远程下发
    │
projectSettings   ←── .claude/settings.json（项目级）
    │
localSettings     ←── .claude/settings.local.json（本地）
    │
userSettings      ←── ~/.claude/settings.json（用户级）
    │
cliArg            ←── CLI 参数
    │
session           ←── 当前会话临时规则
    │
command           ←── 命令级规则（最低优先级）
```

### Shell 规则匹配

Shell 命令使用三种匹配模式：

```typescript
type ShellPermissionRule =
  | { type: 'exact'; command: string }    // 精确匹配
  | { type: 'prefix'; prefix: string }    // 前缀匹配
  | { type: 'wildcard'; pattern: string } // 通配符匹配
```

#### 示例

```json
{
  "allow": [
    { "toolName": "Bash", "ruleContent": "git status" },       // 精确
    { "toolName": "Bash", "ruleContent": "npm:*" },            // 前缀
    { "toolName": "Bash", "ruleContent": "git *" },            // 通配符
    { "toolName": "FileEdit" },                                 // 所有编辑
    { "toolName": "mcp__myserver", "ruleContent": "tool1" }    // MCP工具
  ],
  "deny": [
    { "toolName": "Bash", "ruleContent": "rm -rf *" },
    { "toolName": "WebFetch" }
  ]
}
```

## 权限检查流程

### 决策流程图

```
工具调用请求
     │
     ▼
┌─── deny 规则检查 ───────────────────────┐
│  遍历所有 deny 规则                      │
│  如果匹配 → 返回 { behavior: 'deny' }   │
└──────┬──────────────────────────────────┘
       │ 未匹配
       ▼
┌─── 模式检查 ────────────────────────────┐
│  plan 模式 + 非只读工具                  │
│  → 返回 { behavior: 'deny' }            │
│                                         │
│  bypass 模式                            │
│  → 返回 { behavior: 'allow' }           │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─── allow 规则检查 ──────────────────────┐
│  遍历所有 allow 规则                     │
│  如果匹配 → 返回 { behavior: 'allow' }  │
└──────┬──────────────────────────────────┘
       │ 未匹配
       ▼
┌─── 工具自身权限检查 ────────────────────┐
│  tool.checkPermissions(input, context)   │
│  各工具自定义逻辑                        │
└──────┬──────────────────────────────────┘
       │
       ▼
┌─── 最终决策 ────────────────────────────┐
│  default / acceptEdits:                  │
│    → { behavior: 'ask', suggestions }    │
│                                         │
│  auto:                                  │
│    → 调用 AI 分类器判断                  │
└─────────────────────────────────────────┘
```

### 权限决策类型

```typescript
// 允许
type PermissionAllowDecision<Input> = {
  behavior: 'allow'
  updatedInput?: Input         // 可修改输入
  userModified?: boolean       // 用户是否修改了输入
  decisionReason?: PermissionDecisionReason
  acceptFeedback?: string      // 用户反馈
  contentBlocks?: ContentBlockParam[]  // 附加内容
}

// 询问用户
type PermissionAskDecision<Input> = {
  behavior: 'ask'
  message: string              // 显示给用户的信息
  updatedInput?: Input
  suggestions?: PermissionUpdate[]  // 权限建议（如 "始终允许"）
  blockedPath?: string         // 被阻止的路径
  pendingClassifierCheck?: PendingClassifierCheck  // 异步分类
}

// 拒绝
type PermissionDenyDecision = {
  behavior: 'deny'
  message: string
  decisionReason: PermissionDecisionReason
}
```

### 决策原因

```typescript
type PermissionDecisionReason =
  | { type: 'rule'; rule: PermissionRule }       // 匹配到规则
  | { type: 'mode'; mode: PermissionMode }       // 模式限制
  | { type: 'hook'; hookName: string }           // Hook 决策
  | { type: 'classifier'; classifier: string }   // AI 分类器
  | { type: 'sandboxOverride' }                  // 沙箱覆盖
  | { type: 'asyncAgent'; reason: string }       // 异步 Agent
  | { type: 'workingDir'; reason: string }       // 工作目录限制
  | { type: 'safetyCheck'; reason: string }      // 安全检查
  | { type: 'subcommandResults'; ... }           // 子命令结果
  | { type: 'other'; reason: string }            // 其他
```

### 路径验证

```typescript
function isPathAllowed(resolvedPath, context, operationType): PermissionResult {
  // 1. 检查 deny 规则
  // 2. 检查是否是内部可编辑路径（.claude/ 中特定文件）
  // 3. 安全检查（Windows 系统路径、危险文件等）

  // 6. 检查沙箱写允许列表

```

### 路径安全特性

- **Windows 系统路径保护**：检测并阻止修改系统文件
- **Tilde 展开**：`~/file` → `/home/user/file`
- **Glob 基目录提取**：从 `**/*.ts` 提取有效目录
- **大小写归一化**：跨平台安全比较
- **技能文件域限制**：技能只能编辑其目录范围内的文件

## Auto 模式分类器

### 工作原理

Auto 模式使用 AI 分类器判断操作是否安全：

```
工具调用请求
     │
     ▼
formatActionForClassifier()  ←── 格式化操作描述
     │
     ▼
classifyYoloAction()  ←── AI 分类
     │
     ├─ 安全 → allow
     ├─ 危险 → deny
     └─ 不确定 → ask
```

### 分类器配置

```typescript
type AutoModeRules = {
  allow: string[]        // 始终允许的模式
  soft_deny: string[]    // 软拒绝（可覆盖）
  environment: string[]  // 环境描述
}
```

### 拒绝跟踪

```typescript
const DENIAL_LIMITS = {
  maxConsecutive: 3,    // 连续拒绝 3 次后回退到提示模式
  maxTotal: 20,         // 总拒绝 20 次后回退到提示模式
}
```

防止分类器过度拒绝导致 Claude 无法工作。

## 权限解释器

## 安全设计原则

1. **默认安全** — 默认模式下所有操作都需要确认
2. **最小权限** — 只授予完成任务所需的最小权限
3. **分层控制** — 企业策略 > 项目 > 用户 > 会话
4. **危险文件保护** — 系统配置文件和 Git 内部文件特殊保护
5. **路径沙箱** — 操作限制在工作目录范围内
6. **拒绝回退** — 自动模式连续拒绝时回退到人工确认
7. **审计追踪** — 所有权限决策都记录原因
