# 03 - 启动流程

## 概述

Claude Code 的启动是一个精心编排的多阶段初始化过程，包含并行预取、特性开关加载、插件/技能初始化、MCP 配置、会话恢复等步骤。

## 阶段一：CLI 入口 (`main.tsx`)

### 启动时序

```
[0ms]   开始性能分析计时
        ↓
[~5ms]  并行预取（非阻塞）
        ├── startMdmRawRead()    — MDM 设置读取（macOS: plutil / Windows: 注册表）
        ├── startKeychainPrefetch() — macOS Keychain 预取（OAuth + API Key）
        └── GrowthBook 特性开关初始化
        ↓
[~20ms] Commander.js CLI 参数解析
        ├── 版本号、帮助信息
        ├── --model, --permission-mode, --verbose 等标志
        ├── -e KEY=value 环境变量注入
        └── 子命令路由（mcp, sdk 等）
        ↓
[~30ms] 特性开关初始化
        ├── 从 bundle 加载编译时标志
        └── 从 GrowthBook 加载运行时标志
        ↓
[~40ms] 工具权限上下文构建
        ├── 从 CLI 标志解析
        └── 从配置文件加载
        ↓
[~50ms] 插件初始化
        └── initBundledPlugins()
        ↓
[~60ms] 技能初始化
        └── initBundledSkills()
        ↓
[~80ms] MCP 配置加载
        ├── ~/.claude/mcp.json        （用户配置）
        ├── 企业策略配置              （MDM）
        ├── Claude AI MCP 配置        （远程）
        └── 服务器去重
        ↓
[~100ms] Bootstrap 数据获取
        ├── fetchBootstrapData()     — 用户/组织信息
        ├── checkQuotaStatus()       — Token 配额检查
        └── prefetchAllMcpResources() — MCP 资源预缓存
        ↓
[~150ms] 会话恢复（如有）
        ├── loadConversationForResume() — 恢复之前的对话
        ├── getInitialAdvisorSetting()  — AI 顾问开关
        └── getInitialEffortSetting()   — 任务复杂度设置
        ↓
[~200ms] 渲染 & 启动 REPL
        └── Ink render → 交互式会话开始
```

### 并行预取优化

启动过程中多个 I/O 操作并行执行以加速：

```typescript
// 这些操作同时启动，不互相等待
startMdmRawRead()          // macOS: plutil 读取 plist
                           // Windows: reg query HKCU
startKeychainPrefetch()     // macOS Keychain 查询
                           // 首次 ~20-50ms，后续缓存命中
growthbookInit()           // 特性开关配置拉取
```

## 阶段二：会话初始化 (`setup.ts`)

### 初始化检查

```
Node.js 版本检查 (>= 18)
  ↓
会话管理
  ├── 如提供自定义 session ID → 切换
  └── 初始化 Session Memory 服务
  ↓
UDS 消息服务器（团队/Swarm 模式）
  ├── 启动 Unix Domain Socket 服务器
  ├── 环境变量: CLAUDE_CODE_MESSAGING_SOCKET
  └── 仅 macOS/Linux
  ↓
工作目录设置
  ├── 记录 project root 和原始 CWD
  ├── 创建 Git Worktree（如启用）
  └── 初始化 tmux 会话
  ↓
Hook 系统初始化
  ├── 捕获 hooks 配置快照
  └── 启动文件变更监听器
  ↓
终端备份
  └── iTerm2 / Apple Terminal 状态保存
```

## 阶段三：异步初始化 (`entrypoints/init.ts`)

```
性能分析采样点
  ↓
配置验证
  ├── 合并多源设置
  └── Zod schema 校验
  ↓
遥测设置
  ├── OpenTelemetry 初始化
  ├── Datadog 集成
  └── 事件日志器创建
  ↓
API 客户端创建
  ├── API Key / OAuth Token 解析
  ├── 提供者检测（直连 / Bedrock / Vertex / Foundry）
  └── 超时 & 重试配置
```

## 入口点路由

### 多入口支持

| 入口点 | 文件 | 说明 |
|--------|------|------|
| CLI | `entrypoints/cli.tsx` | 标准终端交互模式 |
| MCP Server | `entrypoints/mcp.ts` | 作为 MCP 服务器运行 |
| SDK | `entrypoints/sdk/` | SDK 集成接口 |

### 命令行参数

```bash
claude [options] [prompt]

选项:
  --model <model>           指定模型
  --permission-mode <mode>  权限模式 (default/acceptEdits/plan/auto/bypass)
  --verbose                 详细输出
  --bare / --simple         简版模式（仅 Bash/Read/Edit）
  --resume <session-id>     恢复会话
  -e KEY=value              注入环境变量
  --flag-settings-path      外部设置文件
  --mcp-config              MCP 配置文件路径
```

## 启动性能优化策略

### 1. 并行预取
```
MDM 读取 ──────┐
Keychain 预取 ──┼── 并行 → 等待全部完成
GrowthBook ────┘
```

### 2. 延迟加载
```typescript
// 大型模块仅在需要时加载
const AgentTool = require('./tools/AgentTool').AgentTool;

// 命令使用 load() 延迟
{ type: 'local-jsx', load: () => import('./commands/config') }
```

### 3. 编译时消除
```typescript
// feature() 在构建时决定，无运行时开销
if (feature('BUDDY')) { /* 外部构建时完全移除 */ }
```

### 4. 缓存利用
```typescript
// 设置缓存 — 避免重复读取文件
settingsCache: { session, file }

// Keychain 缓存 — 首次 20-50ms，后续 0ms
startKeychainPrefetch() // 提前开始预取

// MCP 资源缓存
prefetchAllMcpResources()
```

## 会话恢复机制

当使用 `--resume` 参数时：

```
1. 加载会话文件
   └── loadConversationForResume(sessionId)

2. 恢复状态
   ├── 消息历史
   ├── 工具权限快照
   ├── 工作目录
   └── 成本追踪

3. 重建上下文
   ├── 重连 MCP 服务器
   ├── 恢复 Hook 配置
   └── 重建文件监听

4. 继续交互
   └── Ink render → 从上次中断处继续
```
