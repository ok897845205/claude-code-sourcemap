# 07 - 服务层

## 概述

服务层位于 `services/` 目录下，是 Claude Code 的**基础设施层**，提供 API 通信、MCP 集成、分析遥测、OAuth 认证、会话记忆等核心能力。

### 四种 Provider

#### ① Direct API（直接 API）

```typescript
// 标准 Anthropic API
const client = new Anthropic({
  apiKey: ANTHROPIC_API_KEY,
  baseURL: ANTHROPIC_BASE_URL,    // 可自定义
  defaultHeaders: {
    'x-app': 'cli',
    'User-Agent': getUserAgent(),
    'X-Claude-Code-Session-Id': getSessionId(),
  },
})
```

支持的认证方式：
- `ANTHROPIC_API_KEY` — 标准 API Key
- OAuth Token — claude.ai 订阅者
- `ANTHROPIC_AUTH_TOKEN` — 辅助认证
- `API_KEY_HELPER` — 外部脚本获取 Key

#### ② AWS Bedrock

```typescript
// 条件：CLAUDE_CODE_USE_BEDROCK=true
const client = new AnthropicBedrock({
  awsAccessKey: credentials.accessKeyId,
  awsSecretKey: credentials.secretAccessKey,
  awsSessionToken: credentials.sessionToken,
  awsRegion: AWS_REGION,
})
```

特性：
- 支持 AWS 凭证自动刷新
- `AWS_BEARER_TOKEN_BEDROCK` API Key 认证
- 小模型区域分离（`ANTHROPIC_SMALL_FAST_MODEL_AWS_REGION`）

#### ③ Google Vertex AI

```typescript
// 条件：CLAUDE_CODE_USE_VERTEX=true
const client = new AnthropicVertex({
  projectId: CLOUD_ML_PROJECT_ID,
  region: CLOUD_ML_REGION,
})
```

特性：
- `google-auth-library` 认证
- 模型特定区域变量（`VERTEX_REGION_CLAUDE_*`）
- GCP 凭证自动刷新

#### ④ Azure Foundry

```typescript
// 条件：CLAUDE_CODE_USE_FOUNDRY=true
const client = new AnthropicFoundry({
  resourceName: ANTHROPIC_FOUNDRY_RESOURCE,
  apiKey: ANTHROPIC_FOUNDRY_API_KEY,   // 或
  azureAdToken: DefaultAzureCredential, // Azure AD
})
```

## MCP 服务

### 架构概览

```
┌─────────────────────────────────┐
│          MCP 配置管理            │
│  config.ts — 多作用域配置合并     │
│                                 │
│  作用域：                        │
│  local → user → project →       │
│  dynamic → enterprise →         │
│  claudeai → managed             │
└──────────┬──────────────────────┘
           │
┌──────────▼──────────────────────┐
│       MCPConnectionManager       │
│  React Context Provider          │
│  管理所有 MCP 服务器连接          │
│                                 │
│  状态：                          │
│  connected │ failed │           │
│  needs-auth │ pending │         │
│  disabled                       │
└──────────┬──────────────────────┘
           │
┌──────────▼──────────────────────┐
│          MCP Client              │
│  @modelcontextprotocol/sdk       │
│                                 │
│  传输层：                        │
│  stdio │ sse │ http │ ws │ sdk  │
└─────────────────────────────────┘
```

### 传输类型

| 传输 | 说明 | 配置示例 |
|------|------|---------|
| `stdio` | 本地进程通信 | `{ command: "node", args: ["server.js"] }` |
| `sse` | Server-Sent Events | `{ type: "sse", url: "https://..." }` |
| `sse-ide` | IDE 内部 SSE | `{ type: "sse-ide", url: "...", ideName: "vscode" }` |
| `http` | HTTP 请求 | `{ type: "http", url: "https://..." }` |
| `ws` | WebSocket | `{ type: "ws", url: "wss://..." }` |
| `sdk` | SDK 内置 | `{ type: "sdk", name: "builtin" }` |

### MCP 连接状态

```typescript
type MCPServerConnection =
  | { type: 'connected'; client; capabilities; instructions }
  | { type: 'failed'; error: string }
  | { type: 'needs-auth' }          // 需要 OAuth 认证
  | { type: 'pending'; reconnectAttempt: number }
  | { type: 'disabled' }
```

## OAuth 服务

### 认证流程

```
┌──────────┐     ┌──────────────┐     ┌────────────┐
│  Claude   │────►│ OAuth        │────►│ Anthropic  │
│  Code     │     │ Service      │     │ Auth Server│
└──────────┘     └──────────────┘     └────────────┘
     │                 │                      │
     │   1. 生成 PKCE  │                      │
     │   (code_verifier│+ code_challenge)      │
     │                 │                      │
     │   2. 构建授权 URL│                      │
     │   (state + challenge)                   │
     │                 │                      │
     │   3. 打开浏览器  │                      │
     │   ─────────────►│   4. 用户授权         │
     │                 │   ─────────────────►  │
     │                 │                      │
     │                 │   5. 回调 auth code    │
     │                 │   ◄─────────────────  │
     │                 │                      │
     │   6. 交换 token │                      │
     │   (code + verifier)                     │
     │                 │   ─────────────────►  │
     │                 │                      │
     │                 │   7. 返回 token        │
     │   ◄─────────────│   ◄─────────────────  │
     │                 │                      │
     │   8. 获取 profile                       │
     │   ─────────────►│   ─────────────────►  │
     │   ◄─────────────│   ◄─────────────────  │
└──────────┘     └──────────────┘     └────────────┘
```

### 核心组件

```typescript
class OAuthService {
  constructor()  // 生成 PKCE codeVerifier

  async startOAuthFlow(
    authURLHandler,      // URL 处理回调
    options?: {
      loginWithClaudeAi?: boolean,   // 使用 claude.ai 登录
      inferenceOnly?: boolean,       // 仅推理权限
      expiresIn?: number,            // Token 有效期
      orgUUID?: string,              // 组织 ID
      loginHint?: string,            // 登录提示
      loginMethod?: string,          // 登录方式
      skipBrowserOpen?: boolean,     // 跳过浏览器
    }
  ): Promise<OAuthTokens>
}
```

## Session Memory 会话记忆

### 工作原理

```
对话进行中
     │
     ├─► hasMetInitializationThreshold()  ← 首次触发条件
     │   （足够多的对话轮次）
     │
     ├─► runForkedAgent()  ← 分叉子 Agent
     │   使用 buildSessionMemoryUpdatePrompt()
     │   提取对话关键信息
     │
     ├─► 写入 Markdown 文件
     │   存储在 getSessionMemoryPath()
     │
     ├─► hasMetUpdateThreshold()  ← 后续更新条件
     │   （更多轮次后增量更新）
     │
     └─► 循环...
```

### 特性

- **GrowthBook 门控** — `tengu_session_memory`
- **远程配置** — `tengu_sm_config`（动态调整阈值）
- **后台运行** — 通过 forked subagent 不中断主对话
- **增量更新** — 首次创建后定期增量更新
