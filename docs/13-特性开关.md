# 13 - 特性开关

## 概述

Claude Code 使用**双层特性开关系统**：编译时 `feature()` 函数（死代码消除）和运行时 GrowthBook SDK（动态配置）。

## 编译时特性开关：feature()

### 工作原理

```typescript
import { feature } from 'bun:bundle'

// Bun 构建时求值，生成 true/false 常量
if (feature('VOICE_MODE')) {
  // 只在启用 VOICE_MODE 的构建中存在
  registerVoiceCommand()
}

// 外部构建中，禁用的特性代码被物理移除（死代码消除）
```

### 特性

- **编译时求值** — Bun bundler 在构建时计算
- **死代码消除** — 禁用的特性代码不会出现在最终产物中
- **零运行时开销** — 不需要运行时条件判断
- **安全** — 内部特性在外部构建中完全不存在

## 运行时特性开关：GrowthBook

### 集成方式

```typescript
// services/analytics/growthbook.ts
// 使用 @growthbook/growthbook SDK，remoteEval: true

// 非阻塞读取（从内存缓存或磁盘）
function getFeatureValue_CACHED_MAY_BE_STALE<T>(
  feature: string,
  defaultValue: T
): T

// 阻塞读取（等待初始化，5秒超时）
function getFeatureValue_DEPRECATED<T>(
  feature: string,
  defaultValue: T
): Promise<T>

// Statsig 迁移门控
function checkGate_CACHED_OR_BLOCKING(
  gate: string
): Promise<boolean>

// 获取所有特性值
function getAllGrowthBookFeatures(): Record<string, unknown>
```

### 用户属性（用于定向投递）

```typescript
type GrowthBookUserAttributes = {
  id: string                    // 用户 ID
  sessionId: string             // 会话 ID
  deviceID: string              // 设备 ID
  platform: 'win32' | 'darwin' | 'linux'
  apiBaseUrlHost?: string       // API 域名
  organizationUUID?: string     // 组织 ID
  accountUUID?: string          // 账户 ID
  userType?: string             // 用户类型
  subscriptionType?: string     // 订阅类型
  rateLimitTier?: string        // 速率限制级别
  firstTokenTime?: number       // 首次 Token 时间
  email?: string                // 邮箱
  appVersion?: string           // 应用版本
}
```

### 覆盖机制（优先级从高到低）

```
1. CLAUDE_INTERNAL_FC_OVERRIDES 环境变量
   └── JSON Map，仅内部用户，最高优先级

2. ~/.claude.json → growthBookOverrides
   └── 通过 /config Gates 标签设置，仅内部用户

3. GrowthBook 远程求值响应
   └── 正常的远程配置

4. 磁盘缓存回退
   └── ~/.claude.json → cachedGrowthBookFeatures
```

### 生命周期

```
启动
 │
 ├─► 初始化 GrowthBook（5秒超时）
 │
 ├─► 处理远程求值结果
 │
 ├─► 缓存到磁盘
 │
 └─► 定期刷新
      ├── 内部用户：20 分钟
      └── 外部用户：6 小时
```

## 编译时 vs 运行时对比

| 特性 | 编译时 (feature()) | 运行时 (GrowthBook) |
|------|-------------------|---------------------|
| 实现 | Bun bundler 宏 | GrowthBook SDK |
| 评估时机 | 构建时 | 运行时 |
| 死代码消除 | ✅ | ❌ |
| 动态切换 | ❌ 需重新构建 | ✅ 可实时切换 |
| 用户定向 | ❌ 全局 | ✅ 按属性定向 |
| A/B 测试 | ❌ | ✅ |
| 性能影响 | 零 | 极低（缓存） |
| 适用场景 | 大型功能模块门控 | 渐进发布、实验 |
